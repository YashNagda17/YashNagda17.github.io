This week, I am starting the GSoC project for integrating MLIR backend with Lfortran. The main goal is to understand the existing lowering flow, note the most important gaps, and prepare a clean foundation for later MLIR implementation. The project has two paths: a native API built with our own implementation for MLIR lowering, and an upstream API built on upstream MLIR. This early work involves creating a stable API Design for future changes in the MLIR backend, and ensuring that the design can work with both native and upstream implementations.

The project is important because a reliable MLIR backend can make LFortran easier to extend with upstream MLIR changes. The stablitity of LFortran MLIR backend with upstream changes, with minimum changes in the codebase is crucial. Also, it gives a performance comparision with existing LFortran backends, and helps to identify the most important gaps in the current implementation. The project also involves a lot of design work, which is important for ensuring that the MLIR backend can be easily extended in the future.
